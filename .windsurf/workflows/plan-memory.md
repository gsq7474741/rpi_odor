# 方案记忆管理工作流

当用户提出一个需要多步骤实现的功能需求时，使用此工作流来管理开发上下文。

## 1. 创建方案记忆

当用户确认方案后，创建一个记忆来记录完整开发方案：

```
使用 create_memory 工具:
- Title: "[功能名称]开发方案"
- Tags: ["development_plan", "feature_plan", "[功能标签]"]
- Content 应包含:
  - 功能模块概述
  - 数据库表设计（带 checkbox）
  - 后端模块结构（带 checkbox）
  - 前端页面结构（带 checkbox）
  - 实现顺序和进度表
```

## 2. 开发过程中更新记忆

每完成一个子模块后，更新记忆进度：

// turbo
```
使用 create_memory 工具 (Action: update):
- 找到对应记忆的 ID
- 将已完成的 checkbox 从 [ ] 改为 [x]
- 更新进度表状态：⏳ 待开始 → 🔄 进行中 → ✅ 已完成
```

## 3. 记忆内容模板

```markdown
# [功能名称]开发方案

## 状态：[规划中/开发中/已完成]

## 一、功能模块
- 模块1描述
- 模块2描述

## 二、数据库表
- [ ] table1 - 描述
- [ ] table2 - 描述

## 三、后端模块
- [ ] proto 定义
- [ ] repository 层
- [ ] gRPC 服务

## 四、前端页面
- [ ] 页面1
- [ ] 组件1

## 五、实现顺序
| 阶段 | 内容 | 状态 |
|-----|------|------|
| 1 | xxx | ⏳ 待开始 |
| 2 | xxx | ⏳ 待开始 |
```

## 4. 查找现有方案记忆

如果用户提到继续某个功能的开发，先查找相关记忆：
- 通过 Tags 搜索: "development_plan", "[功能标签]"
- 读取记忆内容确认当前进度
- 从上次停止的位置继续

## 5. 方案完成后

// turbo
```
更新记忆:
- 状态改为 "已完成"
- 所有 checkbox 标记为 [x]
- 添加完成日期
```
